# Google Workspace Ops

このスキルは、単一ファイルの編集ではなく、共有ドキュメント、スプレッドシート、デックを稼働中のシステムとして運用するためのものです。

## 使うタイミング

- ユーザーが Doc、Sheet、Deck を見つけてその場で更新する必要がある
- Google Drive に保存されたプラン、トラッカー、ノート、顧客リストを統合する
- 共有スプレッドシートのクリーンアップや再構成
- Google Slides デックのインポート、修復、再フォーマット
- 意思決定のための Docs、Sheets、Slides の要約作成

## 推奨ツール面

Google Drive をエントリポイントとし、必要に応じて専門ツールに切り替えます：

- テキスト中心のドキュメントには Google Docs
- 表形式の作業、数式、チャートには Google Sheets
- デック、インポート、テンプレート移行、クリーンアップには Google Slides

ファイル名だけから構造を推測しないでください。まず調査してください。

## ワークフロー

### 1. アセットを見つける

Drive の検索面から開始して以下を特定：

- 正確なファイル
- 関連アセット
- 重複の可能性
- 最近修正されたバージョン

複数のドキュメントが類似している場合、タイトル、所有者、更新日時、フォルダで確認してください。

### 2. 編集前に調査

変更前に：

- 現在の構造を要約
- タブ、見出し、スライド数を特定
- タスクがローカルクリーンアップか構造手術かを判定

安全に作業を実行できる最小のツールを選んでください。

### 3. 精密に編集

- Docs：曖昧な書き直しではなくインデックス認識の編集
- Sheets：明示的なタブと範囲で操作
- Slides：コンテンツ編集を視覚クリーンアップやテンプレート移行と区別

作業がビジュアルやレイアウトに依存する場合は、1 回の巨大な盲目的アップデートではなく、調査と検証で反復してください。

### 4. 稼働システムをクリーンに保つ

ファイルがより広いワークフローの一部である場合、以下も明示してください：

- 重複トラッカー
- 古いデック
- 旧 Doc と正準 Doc
- アセットをアーカイブ、統合、リネームすべきか

## 出力フォーマット

以下を使用：

```text
ASSET
- ファイル名
- タイプ
- なぜこのファイルが正しいか

CURRENT STATE
- 構造サマリ
- 主要な問題やブロッカー

ACTION
- 実施または推奨する編集

FOLLOW-UPS
- アーカイブ / 統合 / 重複整理 / 次に更新すべきファイル
```

## 良いユースケース

- 「アクティブな計画ドキュメントを見つけて凝縮して」
- 「この顧客スプレッドシートをクリーンアップしてチャーンリスクの行を表示して」
- 「このデックを Slides にインポートして見栄えよくして」
- 「古い重複ではなく、現在のトラッカーを見つけて」
