# Social Graph Ranker

ネットワーク認識アウトリーチのための正準の重み付きグラフランキング層。

ユーザーが以下を必要とするときに使用してください：

- 既存のミューチュアルまたはコネクションをイントロ価値でランク付け
- ターゲットリストへのウォームパスをマップ
- 第 1 階と第 2 階のコネクション全体でブリッジ価値を計測
- どのターゲットがウォームイントロに値し、どれが直接コールドアウトリーチかを判断
- `lead-intelligence` や `connections-optimizer` から独立してグラフの数学を理解する

## スタンドアロンで使うタイミング

ユーザーが主にランキングエンジンを求めている場合はこのスキルを選択：

- 「私のネットワークで誰が私を紹介するのに最適な立ち位置？」
- 「これらの人々に到達できる人でミューチュアルをランク付けして」
- 「この ICP に対して私のグラフをマップして」
- 「ブリッジの数学を見せて」

ユーザーが実際には以下を求めている場合は単独で使用しない：

- 完全なリード生成とアウトバウンドシーケンシング → `lead-intelligence` を使用
- ネットワークのプルーニング、リバランス、成長 → `connections-optimizer` を使用

## 入力

収集または推測：

- ターゲットの人、企業、または ICP 定義
- X、LinkedIn、またはその両方でのユーザーの現在のグラフ
- 役割、業界、地理、応答性などの重み付け優先度
- トラバーサル深度と減衰耐性

## コアモデル

与えられるもの：

- `T` = 重み付きターゲットセット
- `M` = あなたの現在のミューチュアル / 直接コネクション
- `d(m, t)` = ミューチュアル `m` からターゲット `t` までの最短ホップ距離
- `w(t)` = シグナルスコアリングからのターゲット重み

ベースブリッジスコア：

```text
B(m) = Σ_{t ∈ T} w(t) · λ^(d(m,t) - 1)
```

ここで：

- `λ` は減衰係数、通常 `0.5`
- 直接パスは完全な価値を寄与
- 追加ホップごとに寄与が半減

第 2 階拡張：

```text
B_ext(m) = B(m) + α · Σ_{m' ∈ N(m) \\ M} Σ_{t ∈ T} w(t) · λ^(d(m',t))
```

ここで：

- `N(m) \\ M` はミューチュアルが知っていてあなたが知らない人々の集合
- `α` は第 2 階リーチの割引、通常 `0.3`

応答調整後の最終ランキング：

```text
R(m) = B_ext(m) · (1 + β · engagement(m))
```

ここで：

- `engagement(m)` は正規化された応答性または関係強度
- `β` はエンゲージメントボーナス、通常 `0.2`

解釈：

- ティア 1：高い `R(m)` と直接ブリッジパス → ウォームイントロ依頼
- ティア 2：中程度の `R(m)` と 1 ホップのブリッジパス → 条件付きイントロ依頼
- ティア 3：低い `R(m)` または実行可能なブリッジなし → 直接アウトリーチまたはフォローギャップ補填

## スコアリングシグナル

グラフトラバーサル前にターゲットを、現在の優先度セットにとって重要なもので重み付け：

- 役割またはタイトルの整合
- 会社または業界のフィット
- 現在の活動と新しさ
- 地理的関連性
- 影響力またはリーチ
- 応答の可能性

トラバーサル後にミューチュアルを以下で重み付け：

- ターゲットセットへの重み付きパス数
- それらのパスの直接性
- 応答性または過去の相互作用履歴
- イントロを行うための文脈的フィット

## ワークフロー

1. 重み付きターゲットセットを構築。
2. X、LinkedIn、またはその両方からユーザーのグラフを取得。
3. 直接ブリッジスコアを計算。
4. 最高価値のミューチュアルに対して第 2 階候補を拡張。
5. `R(m)` でランク付け。
6. 以下を返す：
   - ベストなウォームイントロ依頼
   - 条件付きブリッジパス
   - ウォームパスが存在しないグラフギャップ

## 関連スキル

- `lead-intelligence` — より広いターゲット発見とアウトリーチパイプライン内でこのランキングモデルを使用
- `connections-optimizer` — 維持、プルーン、追加の判断時に同じブリッジロジックを使用
- `brand-voice` — イントロリクエストや直接アウトリーチを起草する前に実行すべき
- `x-api` — X グラフアクセスとオプションの実行パスを提供
