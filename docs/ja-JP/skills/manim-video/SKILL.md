# Manim Video

フォトリアリズムよりも動き、構造、明瞭さが重要な技術解説には Manim を使用してください。

## 起動タイミング

- ユーザーが技術解説アニメーションを求めている
- 概念がグラフ、ワークフロー、アーキテクチャ、指標の推移、システム図である
- ユーザーが X やランディングページ向けの短いプロダクト/ローンチ解説を求めている
- 汎用的に映画的ではなく、精密な印象が欲しい

## ツール要件

- シーンレンダリング用 `manim` CLI
- 必要に応じて後処理用 `ffmpeg`
- 最終組み立てや磨き込み用 `video-editing`
- 最終パッケージに UI の合成、キャプション、追加モーションレイヤーが必要な場合は `remotion-video-creation`

## デフォルト出力

- 短い 16:9 の MP4
- サムネイルまたはポスターフレーム 1 枚
- ストーリーボード + シーンプラン

## ワークフロー

1. 中核となる視覚テーゼを 1 文で定義する。
2. 概念を 3〜6 のシーンに分解する。
3. 各シーンが何を証明するか決める。
4. Manim コードを書く前にシーンアウトラインを書く。
5. 最小の動くバージョンを最初にレンダリング。
6. レンダリングが動いてから、タイポグラフィ、スペーシング、色、ペーシングを締める。
7. 価値が加わる場合のみ、より広いビデオスタックに引き継ぐ。

## シーン計画ルール

- 各シーンは 1 つのことを証明すべき
- 詰め込みすぎた図は避ける
- 画面いっぱいの散らかりより漸進的な開示を優先
- 画面を忙しく保つためではなく、状態変化を説明するために動きを使う
- タイトルカードは短く、意味を凝縮させる

## ネットワークグラフのデフォルト

ソーシャルグラフやネットワーク最適化の解説向け：

- 最適化後のグラフを見せる前に現在のグラフを見せる
- 低シグナルのフォロー雑音を、高シグナルのブリッジと区別する
- ウォームパスのノードとターゲットクラスタを強調する
- 有用であれば、スキルを形作った自己改善の系譜を最終シーンで示す

## レンダリング規約

- ユーザーが縦型を求めない限りデフォルトは 16:9 ランドスケープ
- 低品質のスモークテストレンダリングから開始
- コンポジションとタイミングが安定してから高品質に進める
- ソーシャルサイズで読めるクリーンなサムネイルフレームを 1 枚書き出す

## 再利用可能なスターター

ネットワークグラフ解説のスタート地点として [assets/network_graph_scene.py](../../../../skills/manim-video/assets/network_graph_scene.py) を使用してください。

スモークテスト例：

```bash
manim -ql assets/network_graph_scene.py NetworkGraphExplainer
```

## 出力フォーマット

以下を返します：

- 中核となる視覚テーゼ
- ストーリーボード
- シーンアウトライン
- レンダリングプラン
- 追加の磨き込み推奨事項

## 関連スキル

- `video-editing` — 最終磨き込み
- `remotion-video-creation` — モーション重視の後処理や合成
- `content-engine` — アニメーションが広範なローンチの一部である場合
