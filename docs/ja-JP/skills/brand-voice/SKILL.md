# Brand Voice

実際のソース素材から恒久的なボイスプロファイルを構築し、毎回スタイルを一から導出したり汎用 AI コピーに逃げたりせず、そのプロファイルを至る所で使用してください。

## 起動タイミング

- ユーザーが特定のボイスでコンテンツやアウトリーチを求めている
- X、LinkedIn、メール、ローンチ投稿、スレッド、プロダクトアップデート向けの執筆
- チャネルをまたいで既知の著者のトーンを適応させる
- 既存のコンテンツレーンが一回限りの模倣ではなく再利用可能なスタイルシステムを必要としている

## ソース優先度

利用可能な最も強い実ソースセットを以下の順で使用：

1. 最近のオリジナル X 投稿とスレッド
2. 記事、エッセイ、メモ、ローンチノート、ニュースレター
3. 実際に効果があったアウトバウンドメールや DM
4. プロダクトドキュメント、チェンジログ、README のフレーミング、サイトコピー

汎用的なプラットフォームの見本をソース素材として使用しないでください。

## 収集ワークフロー

1. 可能であれば代表的なサンプルを 5〜20 本集める。
2. ユーザーが古い方が正準と言わない限り、古い素材より最近の素材を優先する。
3. ソースセットに明確な分割がある場合は「公のローンチボイス」と「プライベートな作業ボイス」を分ける。
4. ライブの X アクセスがある場合、執筆前に `x-api` で最近のオリジナル投稿を引く。
5. サイトコピーが重要な場合、現在の ECC ランディングページとリポジトリ/プラグインのフレーミングを含める。

## 抽出すべき項目

- リズムと文の長さ
- 圧縮 vs 説明
- 大文字小文字の規則
- 括弧の使用
- 質問の頻度と目的
- 主張の鋭さ
- 数字、メカニズム、証拠の登場頻度
- トランジションの動作
- 著者が絶対にやらないこと

## 出力契約

再利用可能な `VOICE PROFILE` ブロックを生成し、下流のスキルが直接消費できるようにしてください。スキーマは [references/voice-profile-schema.md](references/voice-profile-schema.md) を使用してください。

プロファイルは構造化され、セッションコンテキストで再利用できる程度に短く保ってください。目的は文学批評ではなく、運用上の再利用です。

## Affaan / ECC のデフォルト

ユーザーが Affaan / ECC のボイスを求め、ライブソースが乏しい場合は、新しいソース素材が上書きしない限りここから開始：

- 直接的、圧縮的、具体的
- 形容詞よりも具体性、メカニズム、証拠、数字
- 括弧は限定、絞り込み、過度の明確化のため
- 大文字小文字は、破る真の理由がない限り慣例通り
- 質問はまれで、釣り餌として使わない
- トーンは鋭く、率直、懐疑的、乾いても良い
- トランジションは努力の跡が感じられ、滑らかに繕われていない

## 絶対禁止

以下のいずれかは削除して書き直し：

- 偽の好奇心フック
- 「X ではなく、Y だけ」
- 「fluff なし」
- 強制的な小文字
- LinkedIn 的ソートリーダーのケイデンス
- 釣り餌の質問
- 「Excited to share（共有できて嬉しい）」
- 汎用的な創業者ジャーニーの埋め草
- ダサい括弧書き

## 永続化ルール

- 同じセッションの関連タスク全体で最新の確認済み `VOICE PROFILE` を再利用する。
- ユーザーが恒久的なアーティファクトを求めたら、要求されたワークスペース位置またはメモリ面にプロファイルを保存する。
- ユーザーが明示的に求めない限り、個人のボイスフィンガープリントを保存するリポジトリ追跡ファイルを作らない。

## 下流での使用

以下の前または内部で使用：

- `content-engine`
- `crosspost`
- `lead-intelligence`
- 記事やローンチ執筆
- X、LinkedIn、メール全体のコールドまたはウォームアウトバウンド

別のスキルが部分的なボイスキャプチャセクションを既に持っている場合、このスキルが正準のソースオブトゥルースです。
