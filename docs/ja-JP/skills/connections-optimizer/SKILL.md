# Connections Optimizer

アウトバウンドを一方向のプロスペクトリストとして扱うのではなく、ユーザーのネットワークを再編成します。

このスキルは以下を扱います：

- X フォロー整理と拡張
- LinkedIn フォローとコネクション分析
- レビュー優先のプルーンキュー
- 追加とフォローの推奨
- ウォームパス特定
- ユーザーの実際のボイスでの Apple Mail、X DM、LinkedIn ドラフト生成

## 起動タイミング

- ユーザーが X フォローをプルーンしたい
- ユーザーがフォロー先や繋がりをリバランスしたい
- ユーザーが「ネットワークを整理」「誰をアンフォローすべき」「誰をフォローすべき」「誰と再接続すべき」と言う
- アウトリーチの質がコールドリスト生成だけでなくネットワーク構造に依存する

## 必須入力

収集または推測：

- 現在の優先事項とアクティブな作業
- ターゲットの役割、業界、地理、またはエコシステム
- プラットフォーム選択：X、LinkedIn、またはその両方
- 手を付けないリスト
- モード：`light-pass`、`default`、または `aggressive`

ユーザーがモードを指定しない場合は `default` を使用。

## ツール要件

### 推奨

- `x-api` — X グラフの調査と最近のアクティビティ
- `lead-intelligence` — ターゲット発見とウォームパスランキング
- `social-graph-ranker` — ユーザーが広範なリードワークフローから独立してブリッジ価値のスコアを望むとき
- Exa / ディープリサーチ — 人物と企業の充実
- `brand-voice` — アウトバウンド起草前

### フォールバック

- LinkedIn 分析と起草のためのブラウザ制御
- API カバレッジが制約されている場合の X 向けブラウザ制御
- メールが正しいチャネルの場合、デスクトップ自動化経由の Apple Mail または Mail.app 起草

## 安全デフォルト

- デフォルトはレビュー優先、盲目的な自動プルーニングは行わない
- X：ユーザーがフォローしているアカウントのみプルーン、フォロワーは絶対に触らない
- LinkedIn：1 次コネクション削除は手動レビュー優先として扱う
- DM、招待、メールを自動送信しない
- 適用ステップの前にランク付けされたアクションプランとドラフトを発行

## プラットフォームルール

### X

- ミューチュアルは一方向フォローよりも粘着性が高い
- 非フォローバックはより積極的にプルーンできる
- 大きく非アクティブまたは消失したアカウントは速やかに表面化
- エンゲージメント、シグナル品質、ブリッジ価値は生のフォロワー数より重要

### LinkedIn

- ユーザーが実際に LinkedIn API アクセスを持つなら API 優先
- API アクセスがないときブラウザワークフローが機能する必要がある
- アウトバウンドフォローと受諾済みの 1 次コネクションを区別
- アウトバウンドフォローはより自由にプルーン可
- 受諾済み 1 次コネクションは自動削除ではなくレビューをデフォルトに

## モード

### `light-pass`

- 高信頼・低価値の一方向フォローのみプルーン
- 残りはレビュー用に表面化
- 小さな追加/フォローリストを生成

### `default`

- バランスの取れたプルーンキュー
- バランスの取れた維持リスト
- ランク付けされた追加/フォローキュー
- 有用な場合はウォームイントロまたは直接アウトリーチを起草

### `aggressive`

- より大きなプルーンキュー
- 古い非フォローバックに対する許容度を下げる
- 適用前のレビューゲートは依然として維持

## スコアリングモデル

以下の正のシグナルを使用：

- 相互性
- 最近のアクティビティ
- 現在の優先事項との整合
- ネットワークブリッジ価値
- 役割の関連性
- 実際のエンゲージメント履歴
- 最近のプレゼンスと応答性

以下の負のシグナルを使用：

- 消失または放棄されたアカウント
- 古い一方向フォロー
- 優先外のトピッククラスタ
- 低価値のノイズ
- 繰り返しの無反応
- より良い代替が多数存在するのにフォローバックなし

ミューチュアルと実際のウォームパスブリッジは、一方向フォローより控えめにペナルティすべきです。

## ワークフロー

1. 優先事項、手を付けない制約、選択プラットフォームを取得。
2. 現在のフォロー / コネクションインベントリを取得。
3. プルーン候補を明示的な理由付きでスコア。
4. 維持候補を明示的な理由付きでスコア。
5. `lead-intelligence` とリサーチ面を使って拡張候補をランク付け。
6. 正しいチャネルを割り当て：
   - ウォームで速いソーシャルタッチポイントには X DM
   - プロフェッショナルグラフ隣接には LinkedIn メッセージ
   - より文脈のあるイントロやアウトリーチには Apple Mail ドラフト
7. メッセージ起草前に `brand-voice` を実行。
8. 任意の適用ステップ前にレビューパックを返す。

## アウトバウンドルール

- デフォルトのメールパスは Apple Mail / Mail.app ドラフト作成。
- 自動送信しない。
- 温度、関連性、文脈の深さに基づいてチャネルを選択。
- メールや無アウトリーチが正しい場合に DM を強制しない。
- ドラフトは自動セールスコピーではなく、ユーザーのように聞こえるべき。

## 関連スキル

- `brand-voice` — 再利用可能なボイスプロファイル
- `social-graph-ranker` — スタンドアロンのブリッジスコアリングとウォームパス計算
- `lead-intelligence` — 重み付きターゲットとウォームパスの発見
- `x-api` — X グラフアクセス、起草、オプションの適用フロー
- `content-engine` — ネットワーク移動の周辺で公開ローンチコンテンツも望むとき
